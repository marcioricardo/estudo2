Testando a Gravação com Commit

Primeiro usar o comando git add + nomedoarquivo

Depois usar git commit -m + "Descrição do que foi alterado entre aspas"

Para enviar as alterações ao GitHub git push origin master

Onde "origin" refere-se ao branch no GitHub e "master" é o nosso repositório local.

Márcio Navegantes


É preciso linkar a branch local com a brach remota, para que o GitHub entenda que estou adicionando conteúdo a uma branch que ele já possui. Por isso preciso de um comando que diga que eu estou lhe enviando uma nova branch que foi criada em meu repositório local. git push -u origin design Para uma branch com nome "design".

Agora posso usar git pulldentro de minha brach local para traser consteúdo que possa ter sido incluído no GitHub por outro usuário



Git branch -r Exibe as branchs remotas que estão mapeadas.

Se usar git pull podemos ver se há alguma branch remota que ainda não foi mapeada



Se eu quiser incluir em meu repositório local uma branch que está no GitHub o comando será: git branch -t design origin/design Com isso crio uma nova branch chamada design que já estará linkada com a branch remota de mesmo nome.



Agora posso usar inclusive git log para ver as alterações realizadas por outros usuários.
